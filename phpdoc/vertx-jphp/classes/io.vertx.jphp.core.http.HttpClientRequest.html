continueHandler()

continueHandler( arg0) : \io\vertx\jphp\core\http\HttpClientRequest

If you send an HTTP request with the header {@code Expect} set to the value {@code 100-continue} and the server responds with an interim HTTP response with a status code of {@code 100} and a continue handler has been set using this method, then the {@code handler} will be called.

You can then continue to write data to the request body and later end it. This is normally used in conjunction with the #sendHead() method to force the request header to be written before the request has ended. $handler [callable] continueHandler($handler)

pushHandler()

pushHandler( arg0) : \io\vertx\jphp\core\http\HttpClientRequest

Set a push handler for this request.<p/>

The handler is called when the client receives a push promise from the server. The handler can be called multiple times, for each push promise.

The handler is called with a read-only HttpClientRequest, the following methods can be called:

  • HttpClientRequest#method()
  • HttpClientRequest#uri()
  • HttpClientRequest#headers()
  • HttpClientRequest#getHost()

In addition the handler should call the HttpClientRequest#handler method to set an handler to process the response.

$handler [callable] the handler

writeCustomFrame()

writeCustomFrame( arg0,  arg1 = null,  arg2 = null) : \io\vertx\jphp\core\http\HttpClientRequest

Like {@link #writeCustomFrame(int, int, Buffer)} but with an {@link HttpFrame}.

$frame [HttpFrame] the frame to write

writeCustomFrame($frame)

Write an HTTP/2 frame to the request, allowing to extend the HTTP/2 protocol.

The frame is sent immediatly and is not subject to flow control.

This method must be called after the request headers have been sent and only for the protocol HTTP/2. The #sendHead(Handler) should be used for this purpose.

$flags [integer] the 8-bit frame flags
